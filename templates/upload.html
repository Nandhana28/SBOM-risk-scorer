Upload an SBOM

Analyze your own software bill of materials. The upload is stored under its own scan and never touches the baseline data.

Accepted formats

CycloneDX JSON — the industry-standard SBOM format emitted by syft, cdxgen, Trivy, etc. Components and their licenses are read automatically.

Native CSV — the schema used by this challenge. Minimum columns: dep_id, application_id, library, version; optional: application_name, license, dependency_type, last_updated, transitive_deps.

Applications referenced by an uploaded SBOM but not otherwise described are given a placeholder profile (proprietary licence model, MEDIUM criticality) so the analysis can run — you can see the result immediately, then refine the application metadata later.
